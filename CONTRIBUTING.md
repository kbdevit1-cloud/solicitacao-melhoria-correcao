# Contribuindo

Obrigado pelo interesse em contribuir com este projeto.

Este projeto deve ser mantido de forma organizada, segura e evolutiva. Toda alteração deve seguir um padrão claro para evitar erros, duplicações ou perda de funcionalidades já existentes.

## Como contribuir

Antes de enviar uma contribuição, verifique se a alteração realmente está relacionada ao objetivo do projeto.

Você pode contribuir com:

- Correção de bugs;
- Melhorias de layout;
- Otimização de desempenho;
- Ajustes de segurança;
- Melhorias na organização do código;
- Novas funcionalidades aprovadas previamente;
- Melhorias na documentação.

## Regras para alterações

Ao alterar o projeto:

1. Não remova funcionalidades existentes sem justificativa.
2. Não altere regras principais do sistema sem aprovação.
3. Não adicione funcionalidades fora do escopo definido.
4. Mantenha o padrão visual do sistema.
5. Evite duplicação de código.
6. Mantenha nomes de arquivos, funções e variáveis claros.
7. Teste a alteração antes de enviar.
8. Documente mudanças importantes.
