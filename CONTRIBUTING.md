# Contribuindo

Obrigado pelo interesse em contribuir com este projeto.

Este projeto deve ser mantido de forma organizada, segura e evolutiva. Toda alteracao deve seguir um padrao claro para evitar erros, duplicacoes ou perda de funcionalidades ja existentes.

## Como contribuir

Antes de enviar uma contribuicao, verifique se a alteracao realmente esta relacionada ao objetivo do projeto.

Voce pode contribuir com:

- Correcao de bugs;
- Melhorias de layout;
- Otimizacao de desempenho;
- Ajustes de seguranca;
- Melhorias na organizacao do codigo;
- Novas funcionalidades aprovadas previamente;
- Melhorias na documentacao.

## Regras para alteracoes

Ao alterar o projeto:

1. Nao remova funcionalidades existentes sem justificativa.
2. Nao altere regras principais do sistema sem aprovacao.
3. Nao adicione funcionalidades fora do escopo definido.
4. Mantenha o padrao visual do sistema.
5. Evite duplicacao de codigo.
6. Mantenha nomes de arquivos, funcoes e variaveis claros.
7. Teste a alteracao antes de enviar.
8. Documente mudancas importantes.

## Padrao de commits

Use mensagens de commit claras, por exemplo:

```txt
corrige validacao de login corporativo
adiciona tela interna de usuarios
melhora carregamento do dashboard
ajusta permissoes de solicitacoes
remove codigo duplicado
```

Evite commits genericos como:

```txt
update
teste
arrumei
alteracoes
```

## Antes de abrir um Pull Request

Antes de enviar uma solicitacao de pull request, confirme se:

- O sistema continua funcionando;
- Nao foram criados erros no login;
- As permissoes continuam corretas;
- O layout permanece consistente;
- Nao foram adicionados arquivos desnecessarios;
- Nao ha senhas, tokens ou dados sensiveis no codigo;
- A alteracao foi testada.

## Relatando problemas

Ao relatar um problema, informe:

- O que aconteceu;
- O que era esperado;
- Como reproduzir o erro;
- Imagens ou prints, se possivel;
- Navegador, sistema operacional ou ambiente usado;
- Mensagem de erro, caso exista.

## Solicitacao de melhorias

Ao sugerir uma melhoria, descreva:

- Qual problema a melhoria resolve;
- Como ela deve funcionar;
- Quem sera beneficiado;
- Se a melhoria afeta alguma regra existente;
- Se existe impacto em permissoes, seguranca ou banco de dados.

## Organizacao

Mantenha o projeto limpo e organizado.

Arquivos temporarios, testes descartaveis, duplicados ou nao utilizados nao devem ser enviados para o repositorio.
